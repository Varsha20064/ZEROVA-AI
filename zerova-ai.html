Five-Stage Detection Pipeline

Each code snippet flows through feature extraction, transformer encoding, graph analysis, and ensemble voting.

📥
Stage 01
Code Ingestion
Raw C/C++ source parsed, tokenized, and normalized for downstream models
🔧
Stage 02
Static Features
20 hand-crafted signals: dangerous APIs, pointer ops, bounds checks, cyclomatic complexity
🧠
Stage 03
CodeBERT Encode
microsoft/codebert-base extracts 768-dim semantic embeddings from token sequences
🕸️
Stage 04
GNN Graph
Token co-occurrence graph processed by 2-layer GCN to capture structural patterns
🗳️
Stage 05
Ensemble Vote
XGBoost · LightGBM · CatBoost combine probabilities into final risk verdict

▸ Vulnerability Coverage

What ZEROVA Detects

Trained on 6,000 annotated C/C++ samples across five CWE categories, plus a safe baseline class.

💥
CWE-119
Buffer Overflow
strcpy, gets, memcpy without bounds — classic stack smashing vectors caught by dangerous-func detection.
💉
CWE-89
SQL Injection
String-concatenated queries and sprintf-built SQL detected via SQL pattern and user-input source signals.
👻
CWE-416
Use-After-Free
malloc/free mismatches and dangling pointer returns caught by alloc-free imbalance tracking.
🔪
CWE-78
Command Injection
system() and popen() with user-controlled inputs identified by dangerous-function + user-input correlation.
📖
CWE-125
Out-of-Bounds Read
Array access without index validation flags — detected via array-access count vs bounds-check ratio.
✅
SAFE
Secure Code Patterns
strncpy, parameterized queries, null checks, bounded reads — patterns that pass the ZEROVA verdict.
